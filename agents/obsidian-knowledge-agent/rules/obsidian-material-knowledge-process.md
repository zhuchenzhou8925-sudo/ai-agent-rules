# Obsidian 资料组知识化处理流程

## 1. 规则定位

本规则用于资料目录、多个文件或资料包的知识化入库。入口流程、确认节点和短报告格式以 `Obsidian-智能归档入口规则.md` 为准。

默认执行 **Obsidian 完整闭环低消耗模式**：成果链路完整，执行说明压缩。

## 2. 完整链路

```text
只读资料分析
-> 资料索引
-> 项目笔记或资料笔记
-> MOC
-> 小批量知识卡片
-> 双链与标签建议
-> 正式库迁移建议
-> 上层 MOC 挂接建议
-> 用户确认
-> 正式迁移
-> reviewed 状态
-> 上层 MOC 挂接
-> 统一标签
-> 清理对应草稿
-> 边界检查
```

除非用户明确要求轻量处理，不得省略 MOC、项目笔记、知识卡片、双链、标签建议、迁移建议和挂接建议。

## 3. 资料分析重点

资料组处理时，上下文用量 优先用于：

- 识别主版本、历史版本、重复资料、外部案例和待确认资料；
- 按“项目资料、产品能力、行业场景、商务资料、汇报材料、可复用知识”六类分类；
- 判断项目主线和资料适用场景；
- 判断正式迁入目录和上层 MOC；
- 提炼高质量知识卡片。

不要把篇幅耗在过程日志和规则复述上。

## 4. 阶段要求

### 4.1 只读资料分析

- 只读取用户授权范围；
- 建立资料清单和分类；
- 标注版本关系、重复资料、不可读文件、待确认事项；
- 不移动、删除、重命名、覆盖或修改原始资料。

### 4.2 资料索引

- 作为资料组入口；
- 记录来源、分类、摘要、关键词、可复用方向、待确认事项；
- 连接后续项目笔记、MOC 和知识卡片。

### 4.3 项目笔记或资料笔记

- 承载资料主线；
- 说明背景、核心判断、关键内容、工作路径、风险边界和可复用表达；
- 列出可拆分知识卡片；
- 给出迁移目录和上层 MOC 挂接建议。

### 4.4 MOC

- 作为主题入口；
- 连接资料索引、项目笔记和知识卡片；
- 检查是否已有同主题 MOC，避免重复创建；
- 给出上层 MOC 挂接建议。

### 4.5 知识卡片

- 首轮默认 3-5 张；
- 每张卡片只表达一个稳定、单一、可复用知识点；
- 必须来源明确、可追踪、可独立引用；
- 不批量拆分低质量卡片。

## 5. 草稿确认

草稿阶段一次性生成完整成果到 `<CODEX_OUTPUT_DIR>`，并输出短报告：

```markdown
## 执行结果
- 模式：完整闭环低消耗
- 新增：
- 修改：
- 建议迁入：
- 建议挂接：
- 原文件：
- blocker：
- 待确认：
```

完成后只询问：

```text
是否确认迁入正式库？
```

## 6. 正式迁移

用户确认后一次性完成：

- 迁入 KnowledgeBase；
- 状态改为 `reviewed`；
- 挂接上层 MOC；
- 补充统一标签；
- 清理 `<CODEX_OUTPUT_DIR>` 对应草稿；
- 做边界检查；
- 输出短报告。

迁移前必须检查目标目录、同名冲突和迁移清单。目标不明确或存在冲突时立即停止。

## 7. 边界检查

边界检查只保留必要结论：

- 原始资料是否未修改；
- 外接硬盘原文件是否未移动；
- 是否未写临时脚本；
- 是否未创建无必要临时文件；
- 是否仅迁移本次确认成果；
- 正式文件是否为 `reviewed`；
- MOC、项目笔记、知识卡片双链是否存在；
- 标签和上层 MOC 挂接是否完成；
- `<CODEX_OUTPUT_DIR>` 对应草稿是否清理。

无异常不展开详细检查过程。

## 8. Blocker

出现 blocker 立即停止，按入口规则输出 Blocker 报告，并写入 `<ERROR_LOG_TEMPLATE>`。

## 9. 禁止事项

- 禁止修改原始资料；
- 禁止移动、删除、重命名外接硬盘原文件；
- 禁止写临时脚本；
- 禁止创建无必要临时文件；
- 禁止未确认迁入正式库；
- 禁止绕过 blocker；
- 禁止重复创建同主题 MOC；
- 禁止把外部案例写成主项目事实；
- 禁止把候选版本写成正式版本；
- 禁止把低价值资料包装成正式知识成果。
