# Obsidian 资料索引生成规则

# 文件用途

本文件用于指导 Codex 将一组资料整理为 Obsidian 可用的资料索引，帮助后续生成或关联多类型主题笔记、项目笔记、MOC、知识卡片和正式入档成果。

# 适用范围

适用于用户明确指定的文件清单、资料目录清单、测试样例清单、项目资料包或多类型资料组。资料索引是资料组入口和来源追踪入口，不替代主题笔记、项目笔记或知识卡片，也不能只做浅层摘要和导航。

# 输入要求

- 必须有明确的资料来源路径、文件清单或用户粘贴内容。
- 必须说明是否为正式资料或测试资料。
- 如为正式资料，只允许只读分析，不得移动、删除、重命名。
- 如资料缺少上下文，必须标注“待确认”。

# 输出要求

- 输出 Markdown 资料索引。
- 必须包含资料来源、分类、摘要、关键词、核心判断、业务启发、适用边界、资料关系、可复用方向、后续知识沉淀方向和待确认事项。
- 必须明确是否为测试输出。
- 用户已授权写入正式 KnowledgeBase 时，合理生成的资料索引可直接入档；测试、预览、异常报告或用户明确要求临时输出时，才写入 `_Codex_Output`。

# 正文结构要求

```markdown
# 资料索引标题

## 1. 来源说明
## 2. 资料清单
## 3. 分类结果
## 4. 资料关系与版本判断
## 5. 重点摘要
## 6. 核心判断
## 7. 业务启发
## 8. 关键词
## 9. 可复用观点
## 10. 适用场景与适用边界
## 11. 后续知识沉淀方向
## 12. 关联笔记与卡片
## 13. 未确认事项
## 14. 自检结果
```

# 双链与标签要求

- 对已生成或拟生成的主题笔记、项目笔记、MOC、知识卡片使用双链。
- 双链格式为 `[[页面名]]`。
- 标签至少包含 `#资料索引`、主题领域标签和状态标签。

# 目录归属规则

- 测试资料索引写入 `<CODEX_OUTPUT_DIR>/_Test/output/`。
- 预览、异常报告、过程检查或用户明确要求的临时资料索引可写入 `<CODEX_OUTPUT_DIR>`。
- 用户授权正式入档时，资料索引写入对应 KnowledgeBase 正式目录。
- `KnowledgeBase/_Codex_Output` 仅作为历史兼容说明，不再作为默认输出区。
- 未获得正式 KnowledgeBase 写入授权时，不得写入正式知识分类目录。

# 禁止事项

- 禁止复制正式资料原文作为索引正文。
- 禁止改写或搬运未授权资料。
- 禁止未授权写入正式知识库。
- 禁止把待确认资料写成已确认事实。
- 禁止写临时脚本。
- 禁止用资料清单、关键词和几段摘要替代资料关系、核心判断、业务启发和沉淀方向。

# 自检清单

| 检查项 | 要求 |
|---|---|
| 来源说明 | 有明确路径或说明 |
| 资料清单 | 至少列出名称、类型、用途 |
| 分类 | 按 `material-type-decision-matrix.md` 识别项目资料、研究/可研、文献、技术、产品、政策标准、商务、方法、混合资料或未来扩展资料 |
| 资料关系 | 标注版本关系、重复关系、主从关系或互补关系 |
| 核心判断 | 至少形成与业务复用相关的明确判断 |
| 业务启发 | 说明资料对方案、可研、汇报、招采或客户沟通的启发 |
| 适用边界 | 标注适用场景、不适用场景和待核验前提 |
| 知识沉淀 | 明确后续主题笔记、项目笔记、MOC、资料索引挂接、知识卡片或可复用表达方向 |
| 风险 | 标注缺失和待确认信息 |
| 入档 | 已获授权才写入正式目录 |

# 测试样例要求

测试资料索引应覆盖多个虚构资料来源，并说明每个资料可用于哪类主题笔记、MOC、知识卡片、资料关系判断或业务表达沉淀。

# 验收标准

- 能清楚说明资料来源和分类。
- 能给出核心判断、业务启发、适用边界和后续知识沉淀方向。
- 能给出后续生成或关联主题笔记、项目笔记、MOC、资料索引挂接和知识卡片的建议。
- 未修改来源资料。
- 未在未授权情况下写入正式资料。
- 测试输出包含 `review` 和 `test` 字段。
