# Obsidian 资料索引生成规则

# 文件用途

本文件用于指导 Codex 将一组资料整理为 Obsidian 可用的资料索引，帮助后续生成项目笔记、MOC、知识卡片或迁移建议。

# 适用范围

适用于用户明确指定的文件清单、资料目录清单、测试样例清单或项目资料包。资料索引只做归纳、标注和入口导航，不替代项目笔记或知识卡片。

# 输入要求

- 必须有明确的资料来源路径、文件清单或用户粘贴内容。
- 必须说明是否为正式资料或测试资料。
- 如为正式资料，只允许只读分析，不得移动、删除、重命名。
- 如资料缺少上下文，必须标注“待确认”。

# 输出要求

- 输出 Markdown 资料索引。
- 必须包含资料来源、分类、摘要、关键词、可复用方向、待确认事项。
- 必须明确是否为测试输出。
- 不得将索引结果直接迁移到正式知识库。

# 正文结构要求

```markdown
# 资料索引标题

## 1. 来源说明
## 2. 资料清单
## 3. 分类结果
## 4. 重点摘要
## 5. 关键词
## 6. 可复用观点
## 7. 适用场景
## 8. 建议生成的笔记
## 9. 未确认事项
## 10. 下一步建议
## 11. 自检结果
```

# 双链与标签要求

- 对已生成或拟生成的项目笔记、MOC、知识卡片使用双链。
- 双链格式为 `[[页面名]]`。
- 标签至少包含 `#资料索引`、主题领域标签和状态标签。

# 目录归属规则

- 测试资料索引写入 `<TEST_WORKSPACE>/output/`。
- 正式资料索引草稿默认写入 `<CODEX_OUTPUT_DIR>`。
- 不得直接写入正式知识分类目录。

# 禁止事项

- 禁止复制正式资料原文作为索引正文。
- 禁止改写或搬运未授权资料。
- 禁止自动迁移正式资料。
- 禁止把待确认资料写成已确认事实。
- 禁止写临时脚本。

# 自检清单

| 检查项 | 要求 |
|---|---|
| 来源说明 | 有明确路径或说明 |
| 资料清单 | 至少列出名称、类型、用途 |
| 分类 | 按项目资料、产品能力、行业场景、商务资料、汇报材料、可复用知识归类 |
| 风险 | 标注缺失和待确认信息 |
| 迁移 | 只建议，不执行 |

# 测试样例要求

测试资料索引应覆盖多个虚构资料来源，并说明每个资料可用于项目笔记、MOC、知识卡片或暂不处理。

# 验收标准

- 能清楚说明资料来源和分类。
- 能给出后续生成笔记建议。
- 未修改来源资料。
- 未迁移正式资料。
- 测试输出包含 `review` 和 `test` 字段。
