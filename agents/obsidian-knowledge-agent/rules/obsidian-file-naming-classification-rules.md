# Obsidian 文件命名与分类规则

# 文件用途

本文件用于统一 Obsidian 相关草稿、测试文件、项目笔记、MOC、知识卡片和资料索引的命名与分类。

# 适用范围

适用于 `<CODEX_OUTPUT_DIR>`、`<TEST_WORKSPACE>/` 以及后续经用户确认后拟迁移到正式知识库的 Markdown 文件。

# 输入要求

- 必须明确文件类型。
- 必须明确主题或项目名称。
- 必须明确生成日期。
- 必须明确是否为测试文件。

# 输出要求

文件名应简洁、稳定、可检索，不使用“临时”“新文件”“资料整理”等空泛名称。

# frontmatter 要求

frontmatter 中的 `title` 应与文件名主题一致，`type` 应与文件类型一致，`status` 默认为 `draft`。

# 正文结构要求

正文标题应与 `title` 保持一致。测试文件必须在正文中说明“测试状态说明”。

# 双链与标签要求

- 双链使用不带后缀的文件主题名。
- 标签优先使用中文业务标签。
- 测试文件必须包含 `#测试`。

# 目录归属规则

| 类型 | 命名格式 | 默认目录 |
|---|---|---|
| 项目笔记 | `项目名称-项目笔记-YYYYMMDD.md` | `<CODEX_OUTPUT_DIR>` |
| MOC | `主题名称-MOC-YYYYMMDD.md` | `<CODEX_OUTPUT_DIR>` |
| 知识卡片 | `知识点名称-知识卡片-YYYYMMDD.md` | `<CODEX_OUTPUT_DIR>` |
| 资料索引 | `主题名称-资料索引-YYYYMMDD.md` | `<CODEX_OUTPUT_DIR>` |
| 测试输入 | `test-用途-编号.md` | `<TEST_WORKSPACE>/input` |
| 测试输出 | `test-类型.md` | `<TEST_WORKSPACE>/output` |
| 测试报告 | `Obsidian-规则闭环测试报告.md` | `<TEST_WORKSPACE>/report` |

# 禁止事项

- 禁止使用含糊文件名。
- 禁止直接使用外部资料原文件名作为知识库文件名。
- 禁止批量重命名正式文件。
- 禁止覆盖已有文件。
- 禁止将测试文件命名为正式成果文件。

# 自检清单

| 检查项 | 要求 |
|---|---|
| 文件名 | 清晰、稳定、可检索 |
| 日期 | 正式草稿建议带日期 |
| 类型 | 与内容一致 |
| 测试标识 | 测试文件必须明确 |
| 目录 | 不得写入正式目录 |

# 测试样例要求

测试文件使用固定名称，便于重复检查：`test-project-material-001.md`、`test-project-note.md` 等。

# 验收标准

- 文件名能判断内容类型和用途。
- 文件目录符合授权范围。
- 测试文件不会与正式知识库文件混淆。
