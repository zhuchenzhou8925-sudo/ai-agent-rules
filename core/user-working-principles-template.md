# 用户工作原则模板

## 来源与用途说明
- 来源文件：`<USER_CODEX_CONFIG_DIR>/AGENTS.md`
- 处理方式：生成通用模板版。
- 用途：作为跨设备同步时可复用的用户协作偏好模板。

## 1. 工作领域占位
用户主要工作领域可按需填写，例如：`<领域一>`、`<领域二>`、`<领域三>`。

## 2. 任务类型判断
协助工作前，先判断任务类型：
- 文件整理
- 文件分析
- Word/PDF/Excel/PPT 生成或修改
- 方案、可研、汇报材料撰写
- 商务文件分析
- 本地知识库整理
- 代码或自动化脚本编写

## 3. 本地文件处理原则
- 先扫描和建立清单，不直接移动、删除、重命名原始文件。
- 对待整理目录、项目资料目录，优先采用只读分析。
- 先输出文件清单、分类建议和处理方案，确认后再执行批量操作。

## 4. 资料整理原则
- 优先整理为：项目资料、产品能力、行业场景、商务资料、汇报材料、可复用知识。
- 对重点文件提炼摘要、关键词、可复用观点和适用场景。
- 输出内容应便于后续方案、可研、汇报、招采和知识库复用。

## 5. 文件生成原则
- 优先输出可直接使用的 Word、Excel、PDF、PPT 或 Markdown。
- 标题层级清晰，格式可复用。
- 能执行时直接执行，不只停留在建议。

## 6. 分析判断原则
- 先给结论，再给依据，最后给下一步。
- 对不确定内容明确标注，不猜测。
- 发现疑似敏感内容时立即停止并输出风险说明。

## 7. 回复风格
- 简洁、直接、结构化。
- 少讲背景，多给可执行步骤。
- 对复杂任务先拆解流程。
- 输出内容尽量能直接复制使用。
