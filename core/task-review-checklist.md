# 任务复核清单

## 上传或提交前检查
- [ ] 是否只处理用户确认的目录和文件。
- [ ] 是否没有复制真实项目资料。
- [ ] 是否没有复制客户资料、合同、报价、招采文件。
- [ ] 是否没有 `.env`、Token、密钥、密码、SSH 私钥。
- [ ] README 是否包含中文说明。
- [ ] `.gitignore` 是否存在并包含敏感文件排除规则。
- [ ] 是否已输出确认报告。
- [ ] 是否等待用户确认。

## 执行前智能判断检查
- [ ] 是否已自动判断修改类型：小修改 / 中等修改 / 大修改 / 高风险修改 / 禁止执行。
- [ ] 是否已自动判断风险等级：低 / 中 / 高 / 禁止。
- [ ] 是否已说明判断依据。
- [ ] 是否已判断是否需要 `draft/*` 分支。
- [ ] 是否已判断是否需要 PR。
- [ ] 是否已判断是否存在 blocker。
- [ ] 是否已判断是否影响 `main` 稳定版本。
- [ ] 是否已判断是否影响原始工作区。
- [ ] 是否已判断是否影响 Obsidian 正式库。
- [ ] 是否已判断是否涉及删除、覆盖、批量处理或重命名。
- [ ] 是否已判断是否需要 `backup/*` 分支。

## 用户确认简化检查
- [ ] 是否已用中文解释修改内容、影响范围、风险等级和回滚方式。
- [ ] 是否已把技术判断转换为用户可确认的选项。
- [ ] 是否明确用户只需确认：同意执行 / 暂停 / 改为 draft 分支 / 改为只检查不修改 / 发给 ChatGPT 判断。
- [ ] 如无法判断风险，是否已生成“待 ChatGPT 协助判断的问题包”并暂停。
