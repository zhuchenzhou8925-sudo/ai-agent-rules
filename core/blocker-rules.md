# 阻塞处理规则

## 必须暂停的情况
- 无法判断文件是否敏感。
- 用户未确认上传范围。
- Git 状态中出现疑似客户、合同、报价、招采、密钥或凭据。
- GitHub 权限、分支或远程仓库状态不明确。
- 发现真实本地绝对路径、真实日志原文、客户或项目敏感信息、合同、报价、招采内容、Token、密钥或私钥准备进入 GitHub 同步范围。
- 仓库不是预期分支、不是 `main...origin/main` 起点、存在未确认脏文件或未跟踪文件。
- 需要 commit、push、创建 PR、merge 或 force push，但用户尚未在当前确认节点明确授权。
- 需要写入正式 KnowledgeBase、`_Codex_Output`、AI 成果暂存区、原始资料区或其他未授权目录。

## 输出要求
暂停时必须说明当前目标、已发现问题、不确定点、建议处理方式和是否需要用户确认。
