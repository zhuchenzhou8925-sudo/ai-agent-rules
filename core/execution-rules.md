# 通用执行规则

## 目标
规范智能体执行任务时的边界、确认点和交付要求。

## 规则
- 先理解任务目标，再判断任务类型。
- 涉及本地文件时，先扫描和建立清单。
- 涉及修改、复制、上传时，必须先输出确认报告。
- 不确定或高风险内容必须暂停并说明原因。
- 面向非技术用户时，输出应简洁、直接、结构化。

## GitHub 执行边界
- 后续实质性修改默认使用 `draft/*` 分支。
- 未经用户确认，不得直接 push 到 `main`。
- push 前必须输出 GitHub push 前确认报告。
- PR 前必须输出 Pull Request 创建前确认报告。
- 合并 `main` 前必须等待用户确认。
- 工具类修改必须优先支持 dry-run。

## 不确定问题处理模板
如果 Codex 无法判断技术风险，不得自行猜测。必须生成：

```markdown
# 待 ChatGPT 协助判断的问题包
## 1. 当前任务背景
## 2. 当前目标
## 3. 已发现问题
## 4. 不确定点
## 5. 需要 ChatGPT 判断的问题
## 6. Codex 初步建议
## 7. 是否暂停执行
```
