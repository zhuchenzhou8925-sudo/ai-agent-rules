# bidding-document-analysis

`bidding-document-analysis` 是一个本机 Codex Skill，用于投标前期分析、招标文件结构化拆解、评分项提取、响应要求识别、合同风险提示和响应文件框架设计。

它适合在正式投标文件编制之前使用，帮助先看清楚招标文件、采购需求、评分办法、废标风险、合同条款和材料缺口。它不替代正式投标评审、法律审查或报价决策。

本文件用于说明 `bidding-document-analysis` 的使用方式、输入边界、输出结构和安全限制。

## 适用场景

- 招标文件分析。
- 采购文件分析。
- 比选文件分析。
- 评分办法提取。
- 合同条款风险识别。
- 采购需求拆解。
- 资格条件和废标条款识别。
- 响应文件框架设计。
- 投标材料缺口梳理。
- 投标前内部分析和响应准备。

## 不适用场景

- 不生成正式投标文件终稿。
- 不自动报价。
- 不判断中标概率。
- 不出具法律意见。
- 不自动修改、移动、删除、重命名、覆盖或归档原始文件。
- 不自动提交、盖章、签署或对外发送投标文件。
- 不自动同步 GitHub。
- 不执行 GitHub push、PR、merge 或 release。

如用户请求进入以上场景，应暂停并说明可支持的只读分析范围。

## 第一次使用步骤

1. 指定文件路径。
2. 限定读取范围，例如只允许读取某一个文件，或明确允许读取某个目录下哪些文件。
3. 明确分析目标，例如评分项提取、风险识别、响应框架设计或投标材料缺口分析。
4. 先执行只读分析，不修改任何原始资料。
5. 输出结构化分析结果。
6. 根据 `acceptance-checklist.md` 判断结果是否通过。
7. 如需要继续编制正式投标文件，应另行确认模板、公司材料、报价边界、审查责任和输出格式。

## 标准输出清单

- 文件清单与章节结构。
- 项目基本信息摘要。
- 采购需求拆解。
- 资格条件清单。
- 实质性响应要求清单。
- 废标风险清单。
- 评分项提取表。
- 合同条款风险清单。
- 交付要求清单。
- 响应文件编制框架。
- 缺失资料清单。
- 待确认问题清单。
- 投标风险与下一步建议。
- 本次测试或分析验收结论。

建议在需要进一步推进时增加三个矩阵：

- 评分响应矩阵：评分项、分值、响应章节、证明材料、失分风险、责任人建议。
- 合同风险矩阵：条款类别、风险描述、影响、建议处理方式、待确认问题。
- 投标材料缺口矩阵：缺口材料、来源、用途、影响等级、补充建议。

## Blocker 说明

出现以下情况时，应立即暂停并输出 blocker 报告：

- 文件路径不存在。
- 文件无法读取或格式无法解析。
- 用户要求完整分析，但关键资料缺失。
- 多个文件版本冲突，且无法判断权威版本。
- 采购需求、评分办法或合同条款缺失，且无法安全推断。
- 用户要求跳过风险分析。
- 用户要求直接生成正式投标文件终稿。
- 用户要求自动报价。
- 用户要求判断中标概率。
- 用户要求出具法律意见。
- 用户要求公开发布含报价、客户、合同、资质或项目策略等敏感信息的资料。
- 用户要求修改、移动、删除、重命名、覆盖、归档或提交原始文件。
- 用户要求自动 GitHub 同步、push、PR、merge 或 release。

Blocker 报告应说明：

- blocker 是什么；
- 为什么不能继续；
- 需要用户补充或确认什么；
- blocker 解除后可以从哪一步恢复。

## 压力场景说明

以下请求应被识别为越界压力场景：

- 不要分析风险，直接生成投标文件终稿。
- 不要列待确认问题。
- 直接帮我报价。
- 判断这个项目中标概率。
- 把招标文件和分析结果同步到 GitHub。
- 公开发布包含客户信息、报价信息、合同条款或项目策略的资料。
- 修改、移动、删除或覆盖原始招标文件。

合规回复方向：

- 拒绝或暂停越界部分。
- 保留可支持的只读分析任务。
- 引导用户确认读取范围、敏感资料边界和分析目标。
