# Example: Document Output With Image Placeholders

This example shows how the skill handles Word, PPT, Markdown, report, or solution outputs that need diagrams or images.

## User request

```text
Create a PPT outline from the project knowledge base. Include an architecture diagram and a cover visual.
```

## Example output

```markdown
# PPT Outline: Example Project Knowledge Base

## Slide 1: Cover

Title: Example Project Digital Delivery Plan

【Image / Diagram Placeholder: Cover visual】

Usage position:
Slide 1 full-page background or right-side visual area.

Expression purpose:
Create an immediate visual impression of project digitization, platform capability, and delivery confidence.

Core content:
Project site, digital platform dashboard, data flow, team collaboration, and executive reporting screen.

Recommended layout:
16:9 cover visual, full-bleed or right-side hero image.

Style requirements:
Professional, clean, enterprise-facing, restrained technology style, suitable for formal presentation.

AI image prompt:
Create a professional 16:9 presentation cover visual showing a modern project delivery scenario with a digital platform dashboard, structured data flow, collaborative project team, and executive reporting interface. Clean enterprise style, blue-gray palette, formal business presentation mood, no text embedded in the image.

## Slide 2: Project Background

- Source context
- Business challenge
- Delivery goal
- Knowledge base inputs

## Slide 3: Application Architecture

【Image / Diagram Placeholder: Application architecture diagram】

Usage position:
Slide 3 main content area.

Expression purpose:
Explain how source materials, project data, platform modules, knowledge base notes, and reporting outputs connect.

Core content:
Source files, resource index, project notes, knowledge cards, MOC, document outputs, and human review loop.

Recommended layout:
Layered architecture diagram from input to knowledge processing to output.

Style requirements:
Clear diagram style, minimal decorative elements, suitable for project steering committee or internal review.

AI image prompt:
Create a clean 16:9 layered architecture diagram showing source materials flowing into resource index, project notes, knowledge cards, MOC, human review, and final document outputs. Use professional blue, gray, and white colors. Make the diagram structured, readable, and suitable for a formal project presentation. No real company names or sensitive data.

## Slide 4: Knowledge Reuse Path

- Project note captures the main story.
- Resource index preserves traceability.
- Knowledge cards capture reusable insights.
- MOC connects project, topic, and scenario views.
```

## Rule demonstrated

The assistant does not generate actual images. It only provides placeholders, image purpose, content requirements, layout, style, and prompts for a separate image-generation tool.
