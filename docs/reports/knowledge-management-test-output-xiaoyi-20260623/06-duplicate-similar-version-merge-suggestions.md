## 1. 结论

资料组存在明显多版本、多格式、过程稿与成果稿混放情况。正式整理时不建议直接按文件名批量归档，应先确定“最终可用版本”和“历史参考版本”，再决定知识库只保留哪些知识化结果。

## 3. 建议版本命名规则（正式整理前）

```text
产品名-资料类型-年份-版本状态-格式
示例：
晓羿协同平台-产品介绍-2025-发布版.pdf
晓羿协同平台-产品介绍-2025-编辑版.pptx
晓羿协同平台-产品介绍-2024-过程稿.pptx
```

## 4. 不建议自动合并的原因

- 文件中存在商务、账号、证照、开票等敏感类别，具体文件名已占位。
- 多个版本可能分别服务不同业务场景或年份。
- 当前只读取两个优先文件，不能判断所有历史文件的内容差异。
- 部分 PDF/Key/图片类文件需要 OCR 或视觉比对才可判断重复。
