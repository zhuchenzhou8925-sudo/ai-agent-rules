# Codex 执行入口规则

## 基本原则
- 默认使用中文说明修改内容、影响范围、风险等级和回滚方式。
- 未经用户确认，不得 push、创建 PR 或合并 main。
- 不复制、不上传客户资料、合同、报价、招采文件、内部敏感资料和未脱敏资料。
- 复制现有规则文件前，必须先生成候选清单和映射表。

## 本仓库当前状态
本仓库已初始化基础目录，并包含经过脱敏整理的智能体规则副本。

## GitHub 后续维护规则
- `main` 为稳定版本，只存放用户已确认内容。
- 后续实质性修改默认新建 `draft/YYYY-MM-DD-task-name` 分支。
- 重大调整前可创建 `backup/*` 分支。
- 不得直接 push 到 `main`，除非用户明确确认。
- push 前必须输出 GitHub push 前确认报告。
- 创建 PR 前必须输出 Pull Request 创建前确认报告。
- Codex 不得自行合并 PR，合并 `main` 前必须等待用户确认。
- 工具类修改必须优先支持 dry-run。

## 不确定问题处理
如果 Codex 无法判断技术风险，不得自行猜测。必须生成“待 ChatGPT 协助判断的问题包”，并暂停等待用户确认。
